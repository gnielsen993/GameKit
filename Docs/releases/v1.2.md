# v1.2

Date: in progress (2026-05-12 → )
Type: minor

## Summary
- Video Mode milestone — optional layout adaptation that keeps GameDrawer
  playable while a floating PiP video sits on screen. User manually picks
  one of six PiP positions (Large top/bottom + 4 corners); every game
  screen reflows accordingly. Design-first build: Phase 8 locked the
  visual language against real screenshots before any code shipped.

## User-facing changes
- **Phase 9 (Video Mode foundation).** Settings → VIDEO MODE card with
  an Off/On toggle (default Off, persisted across launches). When On,
  a "Video location" link reveals a visual iPhone-outline picker — tap
  any of 6 zones (Large top, Large bottom, Small top-left / top-right /
  bottom-left / bottom-right) to select where your video appears on
  screen. The selected zone highlights with the active theme's accent
  color and shows "Your video will go here". A short paragraph below
  the picker explains that GameDrawer can't detect your video
  automatically — the manual pick keeps the board and controls clear.
- No game layout changes in this phase. Mines, Merge, and Nonogram
  render byte-identical to v1.1 (Phase 10–12 introduce per-game
  reflow).

## Internal changes
- **VideoModeStore** (`Core/VideoModeStore.swift`) — @Observable
  @MainActor user-preference store mirroring the Phase 4 SettingsStore
  pattern verbatim; backs `isEnabled` + `location` to UserDefaults.
- **VideoModeLocation** enum (`Core/VideoModeLocation.swift`) — 6-case
  raw-string-stable vocabulary.
- **VideoCompactControlRow** (`Core/VideoCompactControlRow.swift`) —
  generic `@ViewBuilder` shared compact control row component
  (Back | primary info | picker | secondary info | settings); reads
  Phase 8 token anchors (`theme.radii.button`, `theme.spacing.xl`,
  `theme.spacing.s`); #Preview block shows the 3 game slot mappings
  (Mines / Merge / Nonogram) — Phase 11/12 will adopt it.
- **VideoLocationPickerView** (`Screens/VideoMode/`) — visual iPhone-
  outline picker using GeometryReader + 9:19.5 aspect ratio; 6 zones
  laid out per Phase 8 VIDEO-MODE-LAYOUTS.md vocabulary; A11Y per zone
  (label + isButton trait + selection value).
- **GameKitApp.swift** — adds the VideoModeStore to the 5-store
  injection chain (no impact on existing 4 stores).
- **Localizable.xcstrings** — 12 new `videoMode.*` keys including the
  VIDEO-14 verbatim manual-selection copy.
- **Tests** — 7 new test files (14 @Test funcs) covering store round-
  trip, environment injection, Settings binding, picker tap+a11y,
  catalog key existence, and the SC5 off-state contract.

## Risks / notes
- VideoMode is **Off by default** and no game view reads the store in
  Phase 9, so the off-path is automatic byte-equivalence with v1.1
  (CONTEXT D-15). Phase 11/12 introduce the actual reads.
- Theme audit on the picker sub-screen confirmed legible on Classic
  (Chrome Diner) + at least one Loud preset (Voltage or Dracula) per
  CLAUDE.md §8.12. The selected-zone fill uses
  `theme.colors.accentPrimary.opacity(0.25)` with the "Your video will
  go here" label set to `theme.colors.textPrimary` — pre-empts the
  accent-on-accent legibility failure mode flagged in Phase 9
  research.
- MARKETING_VERSION remains 1.1; bumped to 1.2 in a later Phase 13
  ship plan.
