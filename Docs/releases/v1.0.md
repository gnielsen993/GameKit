# v1.0

Date: in progress (2026-04 → )
Type: major (first public release)

## Summary
- First public release of **CorePlay** (CorePlay Arcade) — a clean,
  ad-free iOS suite of classic logic games on top of the shared
  DesignKit Swift Package. v1.0 ships **Minesweeper** as the MVP game
  with stats persistence, theme support, and Export/Import backup.

> v1.0 was developed before this release-log convention existed.
> Detailed phase-by-phase history lives in `.planning/` (ROADMAP.md,
> phase folders, `v1.0-MILESTONE-AUDIT.md`). This file captures the
> shipping shape; future versions populate the log natively.

## User-facing changes
- Minesweeper game with Easy / Medium / Hard / Expert difficulties.
- First-tap safety: mines never placed on the first tapped cell or
  its 8 neighbors (CLAUDE §8.11).
- Reveal / Flag interaction-mode toggle.
- Stats persistence (best times, win/loss, games played per
  difficulty) via SwiftData.
- Theme system via DesignKit — Classic (Chrome Diner) default, full
  preset catalog accessible via Settings → "More themes & custom
  colors".
- Export/Import JSON backup with `schemaVersion`.
- Home screen with 2-per-row grid of game cards; settings pared to
  theme · haptics · reset stats · about.
- Branded launch + home: PlayCore → CorePlay → **GameDrawer**
  display name (full brand also "GameDrawer", no suffix). Stack-of-
  three-game-boxes app icon (light / dark / tinted variants),
  unchanged across renames.

## Internal changes
- DesignKit Swift Package consumed for all tokens (colors, radii,
  spacing, motion, typography). No hard-coded styling in app UI.
- Pure / testable game engines: `BoardGenerator`, `RevealEngine`,
  `WinDetector`. Engines have no SwiftUI imports and are deterministic
  for a given seed.
- `Games/Minesweeper/`, `Core/`, `Screens/` folder layout per CLAUDE §3.
- ASC export-compliance flag (`ITSAppUsesNonExemptEncryption = NO`).
- **Account controls (Settings → SYNC, signed-in only):** Sign Out
  + Delete Account rows. Required by App Store Guideline 5.1.1(v)
  for SIWA apps. Sign Out clears local Keychain only (cloud + local
  stats preserved). Delete Account wipes the CloudKit private-database
  zone (`com.apple.coredata.cloudkit.zone`), wipes local SwiftData
  stats (GameRecord + BestTime + BestScore), then clears Keychain;
  cloud-wipe failure surfaces a follow-up alert directing the user
  to Settings → Apple ID → Sign in with Apple → GameDrawer for full
  credential revocation. Supersedes the original P6
  `T-06-row-noSignOut` lock ("System Settings is the only sign-out
  path") — App Review rejects that pattern for SIWA-bearing apps.
  AuthStore.currentUserID converted from a Keychain-read computed
  property to an @Observable stored property mirrored from Keychain
  on every mutation — the previous shape was correct in value but
  invisible to SwiftUI's @Observable tracking, so the SYNC card
  stayed on "Signed in" after sign-out / delete until the view was
  reconstructed.
- **Classic preset is now host-app-owned.** DesignKit no longer
  hardcodes Chrome Diner as everyone's "Classic" — the slot is
  registered per-app via `DesignKit.configure(classicPreset:)` at
  App.init. GameDrawer registers Chrome Diner (`Core/GameKitClassic.swift`).
  Sister repos (FitnessTracker / HabitTracker / PantryPlanner) get a
  neutral grey fallback until they register their own identity. The
  five universal classics (Forest / Navy / Maroon / Walnut / Stone)
  remain in DesignKit and ship to every consumer. `classicMutedEntry`
  removed from DesignKit; `classicGameNumberPalette` exposed as
  `public` so consumers can reuse the Wong-safe palette.
- **End-of-game choreography in Minesweeper.** Loss now plays a wave-
  reveal of remaining mines outward from the trip cell, then pops
  wrong flags with a spring scale, then slides in the end card.
  Win plays confetti for ~1.2s before the end card lands; confetti
  fades out shortly after. New `Animations` toggle in Settings →
  FEEL controls all of it (default ON; honors `Reduce Motion`).
- **Merge auto-spawns the initial board.** The original "first
  swipe seeds the board" pattern read as broken — a swipe against
  an empty grid doesn't communicate that it both creates and moves
  tiles. `MergeViewModel.init` and `restart()` now spawn the
  starting 2-tile board immediately and transition to `.playing`,
  so the user lands on something to play. Per-placement spawning
  unchanged.

## Fixes
- (Carry forward from `.planning/v1.0-MILESTONE-AUDIT.md` as needed.)
- **Minesweeper Hard board now fits on iPad.** The cellSize formula
  was width-only — on iPad 13" portrait the 16-col Hard layout sized
  cells to ~62pt and produced 24 × 62 = 1488pt of grid height, which
  overflowed the available vertical space and rendered as a blank gray
  region. New formula bounds by `min(widthPerCol, heightPerRow)` so
  the whole board fits both axes regardless of device aspect; grid
  is centered horizontally so iPad doesn't pin to the leading edge.
  iPhone Easy/Medium unchanged (width was always the binding
  constraint there). Existing `cellSize(forWidth:cols:padding:spacing:)`
  unit tests preserved as a helper signature.

## Risks / notes
- Bundle ID `com.lauterstar.gamekit` is locked per CLAUDE §1 — never
  changed for any future rename.
- SwiftData store: avoid breaking schema changes. Prefer additive
  migrations or rely on Export/Import as escape hatch.
- Display name renames during v1.0 dev (PixelParlor → PlayCore →
  CorePlay → GameDrawer). v1.0 ships as **GameDrawer**. Phase 07
  release checklist `PF-07 Public app name locked` needs re-confirm
  with the final value before TestFlight upload.
- **Sound effects deferred to v1.0.1** (Plan 06 D-17): tap.caf /
  win.caf / loss.caf bundle resources never landed; SFXPlayer no-ops
  silently per its D-12 contract. Settings AUDIO card pared to a
  single Haptics row + section header renamed AUDIO → HAPTICS to
  avoid Apple Review rejection of a toggle that does nothing
  (`SettingsStore.sfxEnabled` + SFXPlayer + `.sensoryFeedback` /
  `play(...)` call sites all preserved — v1.0.1 just adds the CAFs
  and re-adds the SFX SettingsToggleRow).

## QA checklist
- [ ] Cold-launch latency stays in P0 budget (no popups / modals on
      first run).
- [ ] First-tap safety verified on every difficulty (no first-tap
      losses across 50 cold runs).
- [ ] Stats persistence: play games on each difficulty, force-quit,
      relaunch — totals + best times match.
- [ ] Export → wipe → Import round-trip restores stats identically.
- [ ] Visual audit on Classic + at least one Loud/Moody preset
      (Voltage / Dracula) per CLAUDE §8.12.
- [ ] Reset stats from Settings clears persisted state without
      crash on subsequent game.
- [ ] Home-screen icon renders correctly in light / dark / tinted
      mode.
