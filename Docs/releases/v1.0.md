# v1.0

Date: in progress (2026-04 → )
Type: major (first public release)

## Summary
- First public release of **CorePlay** (CorePlay Arcade) — a clean,
  ad-free iOS suite of classic logic games on top of the shared
  DesignKit Swift Package. v1.0 ships **Minesweeper** as the MVP game
  with stats persistence, theme support, and Export/Import backup.

> v1.0 was developed before this release-log convention existed.
> Detailed phase-by-phase history lives in `.planning/` (ROADMAP.md,
> phase folders, `v1.0-MILESTONE-AUDIT.md`). This file captures the
> shipping shape; future versions populate the log natively.

## User-facing changes
- Minesweeper game with Easy / Medium / Hard / Expert difficulties.
- First-tap safety: mines never placed on the first tapped cell or
  its 8 neighbors (CLAUDE §8.11).
- Reveal / Flag interaction-mode toggle.
- Stats persistence (best times, win/loss, games played per
  difficulty) via SwiftData.
- Theme system via DesignKit — Classic (Chrome Diner) default, full
  preset catalog accessible via Settings → "More themes & custom
  colors".
- Export/Import JSON backup with `schemaVersion`.
- Home screen with 2-per-row grid of game cards; settings pared to
  theme · haptics · reset stats · about.
- Branded launch + home: PlayCore → CorePlay → **GameDrawer**
  display name (full brand also "GameDrawer", no suffix). Stack-of-
  three-game-boxes app icon (light / dark / tinted variants),
  unchanged across renames.

## Internal changes
- DesignKit Swift Package consumed for all tokens (colors, radii,
  spacing, motion, typography). No hard-coded styling in app UI.
- Pure / testable game engines: `BoardGenerator`, `RevealEngine`,
  `WinDetector`. Engines have no SwiftUI imports and are deterministic
  for a given seed.
- `Games/Minesweeper/`, `Core/`, `Screens/` folder layout per CLAUDE §3.
- ASC export-compliance flag (`ITSAppUsesNonExemptEncryption = NO`).

## Risks / notes
- Bundle ID `com.lauterstar.gamekit` is locked per CLAUDE §1 — never
  changed for any future rename.
- SwiftData store: avoid breaking schema changes. Prefer additive
  migrations or rely on Export/Import as escape hatch.
- Display name renames during v1.0 dev (PixelParlor → PlayCore →
  CorePlay → GameDrawer). v1.0 ships as **GameDrawer**. Phase 07
  release checklist `PF-07 Public app name locked` needs re-confirm
  with the final value before TestFlight upload.

## QA checklist
- [ ] Cold-launch latency stays in P0 budget (no popups / modals on
      first run).
- [ ] First-tap safety verified on every difficulty (no first-tap
      losses across 50 cold runs).
- [ ] Stats persistence: play games on each difficulty, force-quit,
      relaunch — totals + best times match.
- [ ] Export → wipe → Import round-trip restores stats identically.
- [ ] Visual audit on Classic + at least one Loud/Moody preset
      (Voltage / Dracula) per CLAUDE §8.12.
- [ ] Reset stats from Settings clears persisted state without
      crash on subsequent game.
- [ ] Home-screen icon renders correctly in light / dark / tinted
      mode.
