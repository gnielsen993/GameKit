# v1.0.1

Date: 2026-05-07
Type: patch

## Summary
- Polish pass on Minesweeper end-of-game feel + Merge UX bug fix +
  the DesignKit "Classic" preset slot being made host-app-owned so
  GameDrawer's Chrome Diner identity no longer leaks into sister
  apps on the same package.

## User-facing changes
- **End-of-game choreography in Minesweeper.** Loss now plays a wave-
  reveal of remaining mines outward from the trip cell, then pops
  wrong flags with a spring scale, then slides in the end card. Win
  plays confetti for ~1.2s before the end card lands; confetti fades
  out shortly after. New `Animations` toggle in Settings → FEEL
  (renamed from HAPTICS since it now covers both subtle-feedback
  knobs). Default ON; honors the system `Reduce Motion` setting.

## Internal changes
- **Classic preset is now host-app-owned.** DesignKit no longer
  hardcodes Chrome Diner as everyone's "Classic" — the slot is
  registered per-app via `DesignKit.configure(classicPreset:)` at
  App.init. GameDrawer registers Chrome Diner (`Core/GameKitClassic.swift`).
  Sister repos (FitnessTracker / HabitTracker / PantryPlanner) get a
  neutral grey fallback until they register their own identity. The
  five universal classics (Forest / Navy / Maroon / Walnut / Stone)
  remain in DesignKit and ship to every consumer. `classicMutedEntry`
  removed from DesignKit; `classicGameNumberPalette` exposed as
  `public` so consumers can reuse the Wong-safe palette.

## Fixes
- **Merge auto-spawns the initial board.** The original "first
  swipe seeds the board" pattern read as broken — a swipe against
  an empty grid doesn't communicate that it both creates and moves
  tiles. `MergeViewModel.init` and `restart()` now spawn the
  starting 2-tile board immediately and transition to `.playing`,
  so the user lands on something to play. Per-placement spawning
  unchanged.

## Risks / notes
- Animation Tasks in `MinesweeperGameView.runLossChoreography` /
  `runWinChoreography` are spawned per-phase-change; if the user
  taps Restart mid-cascade the `.idle` arm in `.onChange(of: phase)`
  resets the gates so the next session lands clean. The orphan Tasks
  finish their `Task.sleep` and write into a state that is already
  reset — harmless but worth knowing if the choreography ever grows
  side-effects beyond `@State` flips.
- Sister repos (FitnessTracker / HabitTracker / PantryPlanner)
  consume the new fallback Classic until they migrate. They render
  but look generic; one-line migration is documented in
  `Core/GameKitClassic.swift`.

## QA checklist
- [ ] Lose Minesweeper: bombs cascade outward from trip cell, wrong
      flags pop, end card slides in.
- [ ] Win Minesweeper: confetti plays for ~1.2s before the end card.
- [ ] Toggle Settings → FEEL → Animations off: end cards land
      immediately on win/loss with no pre-roll.
- [ ] Toggle iOS Settings → Accessibility → Reduce Motion on:
      end cards land immediately even with Animations on.
- [ ] Open Merge: board already populated with 2 tiles, no swipe
      needed to start.
- [ ] Restart Merge mid-game: board re-populates with 2 tiles and
      stays in `.playing`.
- [ ] Theme spot-check: Classic + one Loud preset (Voltage / Dracula)
      per CLAUDE §8.12 — confetti and mine-wave colors track the
      active preset.
