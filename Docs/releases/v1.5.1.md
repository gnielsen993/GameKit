# v1.5.1

Date: 2026-07-16
Type: patch

## Summary
- Nonogram interaction reliability patch.
- Polished native launch and branded app-entry experience.

## User-facing changes
- Launch now opens with the GameDrawer mark on a matching native and in-app brand frame, then transitions directly into onboarding or The Drawer without flashing the wrong destination.
- GameDrawer now uses the 9A.2 Video drawer app icon across light, dark, and tinted appearances.
- Nonogram now previews the targeted cell and highlights its row/column hints while the finger is down. Players can slide to correct an adjacent-cell miss before lifting to commit, making 20×20 boards safer in Lives mode.
- The Nonogram grid now uses 85% of its available width instead of 78%, giving every cell a larger physical target—most noticeably on 20×20 puzzles.
- Horizontal Nonogram clues now use explicit centered-dot boundaries (`1·5·11`) so adjacent clues cannot be mistaken for one multi-digit number; the hint rail remains the same size.
- Nonogram clue cross-offs now preserve puzzle information: unique completed run lengths still cross immediately, while repeated values wait for visible progress anchored from either edge.
- Fast Nonogram swipes fill the complete path immediately instead of visibly trailing behind the finger.

## Internal changes
- Moved SwiftData container preparation behind an observable startup controller, with delayed progress feedback, debug launch-path seams, and focused state/UI coverage.
- Added regression coverage for swipe activation thresholds, horizontal/vertical path interpolation, row-hint display/accessibility grammar, and nuanced clue cross-off behavior.

## Fixes
- Replaced the startup-time SwiftData fatal error with a data-safe retry screen when the persistent container cannot open.
- Removed the cell-edge sample rejection that delayed fast-swipe interpolation.
- Unified tap targeting and swipe-to-fill under one board gesture: short touches commit the previewed cell on lift, while longer axis-locked movement becomes a smear.
- Expanded the grid while retaining the existing hint strips; a broader hint-layout redesign remains intentionally out of scope for this patch.

## Risks / notes
- The native launch frame uses a fixed brand background because UIKit renders it before DesignKit and the saved user theme are available; the app crossfades from that frame into the resolved theme.
- The selected icon intentionally uses one unified full-color master across all three iOS appearance slots.
- Nonogram swipe activation now requires 24–32pt of initial travel, leaving room to correct an adjacent target before touch-up; interpolation still backfills every cell crossed before activation.
