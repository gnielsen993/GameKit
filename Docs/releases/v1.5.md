# v1.5

Date: 2026-07-10
Type: minor

## Summary
- Arcade release: adds Stack and Snake, plus an app-wide depth + smoothness style pass.

## User-facing changes
- Stack: new tap-to-drop endless arcade game — stack sliding blocks, trim the overhang, and build a streak to recover width. Real 3D: blocks are length × width × height boxes on a ground plane in a fixed isometric camera; the slide direction alternates every placement (left-right, then front-back), and a sloppy drop trims only the axis the block was travelling along, so the footprint shrinks in both dimensions over a bad run. Pseudo-3D shaded slabs with a palette gradient up the tower, eased camera scroll, tumbling trim FX, perfect-drop pulse rings, and a sky backdrop that shifts with the tower's palette. Uses the active preset's accent palette, persists high score and best perfect streak, has its own Stats section, and fully honors Reduce Motion.
- Snake: new endless arcade game — swipe or tap the always-visible D-pad to steer, eat food to grow and roll the score chip, toggle between toroidal wrap and wall-death mode from the toolbar menu, and beat your high score across endless runs. Head-to-tail palette gradient, eye-dot head, smooth Gaffer interpolation, and a grayscale drain into the game-over banner; Reduce Motion cuts to a clean jump-cut.
- Stack and Snake stats cards lead with a large High Score hero numeral, followed by Average Score and Runs Played — with Best Streak as an additional row for Stack. Empty state reads "No runs yet." This score-based shape makes arcade stats visually distinct from the turn-based win/loss/best-time cards.
- App-wide style pass: a consistent physical-depth layer — info chips, mode pills, number-pad keys, keyboard keys, and word tiles sit on a soft ambient shadow; Merge tiles, Minesweeper's unrevealed cells, and Home's game tiles get a subtle top-light sheen so pressable surfaces read as raised caps; Word Grid tiles glow with the accent color while held under the trace. All lighting is preset-independent (`Core/SurfaceDepth.swift`); boards/wells/revealed cells deliberately stay flat.
- App-wide smoothness pass: Merge tiles actually slide, pop when they merge, and scale in when they spawn (identity-tracked animation); mode pills animate the active segment sliding between choices; score/best/mines/found-word counters roll numerically instead of snapping; Sudoku digits and Five Letter letters pop in on placement; invalid Five Letter guesses shake the row; Word Grid tiles lift under the trace and found words spring into the panel; number pads, keyboards, and Home tiles compress subtly under the finger. Every new motion hard-cuts to instant under Reduce Motion or with animations off, via a shared `feedbackAnimation` gate.
- Video Mode compliance audit (all games): new "necessity principle" (DESIGN §7.7) — chrome only changes from its normal form when the chosen PiP zone actually covers it. Bottom-band and bottom-corner placements keep the nav bar and full-size chrome in Minesweeper, Merge, Nonogram, Stack, FreeCell, Five Letter, and Word Grid; small-corner placements move only the element the PiP covers.
- Snake supports Video Mode (exemption lifted): band placements shrink the board, corner placements move only the covered controls, and the D-pad stays fully tappable in every zone.
- Klondike now reserves space for the video band in large Video Mode zones — previously the PiP sat directly on the tableau. Five Letter and Word Grid handle small PiP corners: bottom corners lift the keyboard / Clear-Submit-Finish row above the PiP; top corners slide only the covered chips and toolbar buttons to the opposite side.
- Minesweeper board no longer sits in an elevated card — the grid floats directly on the screen background (the box read as distracting), with a standard horizontal margin so cells never touch the screen edges.
- Stack perfect-drop tolerance tightened twice from play-testing (0.025 → 0.015 → 0.010): visibly off-center drops no longer snap to center as "perfects".

## Internal changes
- Stack end-to-end assembly: tap to drop, trim overhang, streak-gated width recovery, game-over cinematic (500ms color-drain + VideoModeBanner), gated haptics, scenePhase pause, and score/streak display. Home Stack card routes to the real game (throwaway harness removed).
- StackEngine widened to a 2-axis footprint model (centerX/centerZ + width/depth, alternating slide axis) while staying pure, normalized, and closed-form — the ProMotion-equivalence (SC2) suite passes unchanged, plus a new axis-alternation/per-axis-trim test.
- Snake end-to-end assembly: SnakeGameView wires SnakeBoardCanvas into the ArcadeLoopDriver with swipe + D-pad steering, capacity-2 direction queue with 180° rejection, wall-mode toolbar + abandon alert, 500ms game-over pre-roll, grayscale drain, VideoModeBanner, and three counter-trigger haptics. Phase 15 harness deleted; SC3 gate confirmed zero Core substrate diff.
- Stack stats: replaced the Phase 15 placeholder with a real StackStatsCard showing high score, runs played, and best perfect streak sourced from persisted BestScore rows.

## Fixes
- Fixed the Nonogram freeze when opening a size whose curated puzzle pool was exhausted: procedural 20×20 generation ran up to 60 solver attempts synchronously on the main thread at tap-time, blocking the UI for 30s+. Generation now runs off the main thread behind a themed "Building a fresh puzzle…" loading state, Large fill density was retuned (0.45 → 0.55 — median generation is now ~5ms), and the next procedural puzzle is prefetched in the background. New procedural ids use a `proc2-` prefix; legacy `proc-` ids still reconstruct byte-identical grids.
- Nonogram puzzles are now marked "played" on the first move instead of at pick time — opening the screen, resuming a save, or SwiftUI re-creating the view no longer silently burns entries from the curated rotation.
- Nonogram curated-rotation progress now survives app reinstall: the local "seen puzzles" frontier is rebuilt from synced win records (iCloud sync / stats import) when the game opens, so players aren't re-served puzzles they already solved.
- Minesweeper adjacency numbers are now legible in dark mode on Classic-palette presets: the "4" was near-black (`#212121`, 1.0:1 contrast — invisible on the dark board) and the "8" a dim grey. DesignKit now ships a dark-scheme classic palette (lightened hues, CVD-audited) and Chrome Diner's dark anchors use it.
- Stack tier thickness is now identical in every mode: thickness is tied to the footprint width instead of the canvas height, so normal mode and Video Mode draw the same thin tiers.
- Stack board extends under the home-indicator inset in normal mode too, so the tower reaches the physical bottom of the screen (gameplay framing unchanged — only the pedestal paints lower).
