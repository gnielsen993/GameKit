# v1.1

Date: in progress (2026-05-08 → )
Type: minor

## Summary
- First post-MVP game release. v1.1 adds a new logic puzzle alongside
  Minesweeper and Merge. Game pick is locked in this file once the
  build phase begins.

## User-facing changes
- Nonogram wired into Stats — per-difficulty Games / Wins / Best Time
  rows mirror the Minesweeper card.
- New "Solved puzzles" gallery linked from the Nonogram stats card —
  thumbnail grid grouped by difficulty showing each completed picture
  with its best clear time.
- Home screen redesigned as "The Drawer" — vertical stack of drawer
  rows that hinge open on tap to reveal mode chips inside. Tap a chip
  to drop straight into that mode. Tap anywhere outside an open drawer
  to close.
- Per-game accent colors driven by theme — Forest / Navy / Maroon and
  the other muted presets get a sophisticated palette; Dracula /
  Voltage / Bubblegum and the other bright presets get a saturated
  one. Each game pins to a stable color slot so the cabinet reads as
  three distinct blocks regardless of theme.
- Nonogram board: row hints right-anchored with a dark grid border
  and breathing room between numbers and the grid; cell-edge deadzone
  restored on swipes so glancing fingers no longer mark adjacent rows.
- Merge mode labels cleaned up — "2048 · Classic" / "Infinite ·
  Endless" replaces the previous "Win mode (stop at 2048)" /
  "Infinite mode" copy.

## Internal changes
- `GameRecord.puzzleIdRaw: String?` (additive optional) lets the
  solved-puzzle gallery resolve completed puzzles back to
  `NonogramLibrary` entries. Not exported in v2 envelope (round-trip
  determinism preserved).
- `GameRoute` carries an associated mode/difficulty so the home
  drawer chips deep-link into a specific mode. Each game view's init
  accepts an optional initial mode that threads to its VM.
- DesignKit gains a catalogue palette concept — `PaletteMood`
  (muted / bright) is derived from each preset's existing
  `PresetCategory`. `Theme.catalogueColor(_:)` resolves to the
  appropriate palette so home tiles / drawer faces stay coherent
  across all presets without each preset defining its own palette.

## Fixes
- Nonogram swipe responsiveness — restored the cell-edge deadzone
  that was dropped in 4d9b9da; fast diagonal swipes no longer leak
  into adjacent rows.

## Risks / notes
- Visible bundle-ID surface unchanged; DesignKit API change is
  additive (`Theme.cataloguePalette` defaults to bright when not
  resolved through ThemeResolver).
