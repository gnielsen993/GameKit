# v1.1

Date: in progress (2026-05-08 → )
Type: minor

## Summary
- First post-MVP game release. v1.1 adds a new logic puzzle alongside
  Minesweeper and Merge. Game pick is locked in this file once the
  build phase begins.

## User-facing changes
- Nonogram wired into Stats — per-difficulty Games / Wins / Best Time
  rows mirror the Minesweeper card.
- New "Solved puzzles" gallery linked from the Nonogram stats card —
  thumbnail grid grouped by difficulty showing each completed picture
  with its best clear time.
- Home screen redesigned as "The Drawer" — vertical stack of drawer
  rows that hinge open on tap to reveal mode chips inside. Tap a chip
  to drop straight into that mode. Tap anywhere outside an open drawer
  to close.
- Per-game accent colors driven by theme — Forest / Navy / Maroon and
  the other muted presets get a sophisticated palette; Dracula /
  Voltage / Bubblegum and the other bright presets get a saturated
  one. Each game pins to a stable color slot so the cabinet reads as
  three distinct blocks regardless of theme.
- Nonogram board: row hints right-anchored with a dark grid border
  and breathing room between numbers and the grid; cell-edge deadzone
  restored on swipes so glancing fingers no longer mark adjacent rows.
- Merge mode labels cleaned up — "2048 · Classic" / "Infinite ·
  Endless" replaces the previous "Win mode (stop at 2048)" /
  "Infinite mode" copy.

## Internal changes
- `GameRecord.puzzleIdRaw: String?` (additive optional) lets the
  solved-puzzle gallery resolve completed puzzles back to
  `NonogramLibrary` entries. Not exported in v2 envelope (round-trip
  determinism preserved).
- `GameRoute` carries an associated mode/difficulty so the home
  drawer chips deep-link into a specific mode. Each game view's init
  accepts an optional initial mode that threads to its VM.
- DesignKit gains a catalogue palette concept — `PaletteMood`
  (muted / bright) is derived from each preset's existing
  `PresetCategory`. `Theme.catalogueColor(_:)` resolves to the
  appropriate palette so home tiles / drawer faces stay coherent
  across all presets without each preset defining its own palette.

## Fixes
- Nonogram swipe responsiveness — restored the cell-edge deadzone
  that was dropped in 4d9b9da; fast diagonal swipes no longer leak
  into adjacent rows.
- Phase 9 VideoLocationPickerView UX — replaced the iPhone-outline
  layout that overflowed on iPhone 17 Pro Max (and visually stacked
  small zones underneath large zones, suggesting the wrong mental
  model) with a vertical-stack picker: a Large/Small segmented size
  toggle at the top, then two stacked bands (Top/Bottom). Large mode
  renders each band as one tappable zone; Small mode places two
  corner buttons inside each band — so the small video docks to a
  corner of one of the same two bands, never floats independently.
  Selected zone is bold + "Your video will go here" against a subtle
  accent fill; non-selected zones still read as tap targets. Flipping
  the size toggle preserves the user's Top/Bottom half. Fits without
  scrolling on iPhone 17 Pro Max and SE. 3 additive xcstring keys
  (`videoMode.locationPicker.size{Large,Small,A11yLabel}`); the six
  `videoMode.location.*` zone-label keys are unchanged. All 4 picker
  tests (`VideoLocationPickerViewTests`) GREEN.
- Phase 9 VideoLocationPickerView spatial chrome — restored the
  iPhone-outline frame around the Top/Bottom band stack. The
  11d109a redesign dropped the outline entirely, so the bands read
  as two floating cards with no spatial referent ("zones of what?").
  Wrapped the stack in a stroked `RoundedRectangle`
  (`theme.radii.sheet` corner, `theme.colors.border` stroke, 1.5pt)
  with a `.aspectRatio(9.0 / 19.5, contentMode: .fit)` modifier and
  a token-derived max-width cap (`theme.spacing.xxl * 7`) so the
  outline reads as a modest phone shape and still fits without
  scrolling on iPhone 17 Pro Max and SE. Bands now use
  `frame(maxHeight: .infinity)` to fill the outline's inner area
  evenly (Top = upper half, Bottom = lower half). Outline shape
  carries `.accessibilityHidden(true)` — purely decorative chrome,
  zone labels + container label unchanged. All 4 picker tests still
  GREEN (assertions are behavioral, not layout-coupled).
- Phase 9 VideoLocationPickerView zone footprints — the previous
  rev filled the iPhone outline 50/50 with full-width Top/Bottom
  bands (and a 50/50 split inside each band for Small mode), which
  read as "the band is the zone" instead of "the video goes HERE on
  this part of the screen." Replaced the VStack-of-bands layout
  with a `GeometryReader` + ZStack that sizes each highlighted zone
  to its actual PiP footprint inside the outline: Large mode shows
  two full-width rectangles each ~37% of inner height anchored to
  top/bottom with a ~26% empty gameplay gap between them; Small
  mode shows four corner thumbnails each ~32%×~20% of inner area
  with the rest of the screen empty. All four/two zones remain
  visible simultaneously as highlighted tap targets — selected
  zone bolder fill + thicker accent stroke, non-selected lighter
  fill + thin accent stroke. Footprint ratios are Double literals
  inside proportional math (aspect ratios, not pixel values);
  zone corner radius switched to `theme.radii.button` to match the
  smaller footprint geometry. Selection label uses
  `theme.typography.caption` in Small mode with `minimumScaleFactor`
  so "Your video will go here" fits in the tighter corner zones.
  All 4 `VideoLocationPickerViewTests` still GREEN (behavioral, not
  layout-coupled).
