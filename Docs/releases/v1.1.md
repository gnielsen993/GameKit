Type: minor

## Summary
- First post-MVP game release. v1.1 adds a new logic puzzle alongside
  Minesweeper and Merge. Game pick is locked in this file once the
  build phase begins.

## User-facing changes
- Nonogram wired into Stats ‚Äî per-difficulty Games / Wins / Best Time
  rows mirror the Minesweeper card.
- New "Solved puzzles" gallery linked from the Nonogram stats card ‚Äî
  thumbnail grid grouped by difficulty showing each completed picture
  with its best clear time.
- Home screen redesigned as "The Drawer" ‚Äî vertical stack of drawer
  rows that hinge open on tap to reveal mode chips inside. Tap a chip
  to drop straight into that mode. Tap anywhere outside an open drawer
  to close.
- Per-game accent colors driven by theme ‚Äî Forest / Navy / Maroon and
  the other muted presets get a sophisticated palette; Dracula /
  Voltage / Bubblegum and the other bright presets get a saturated
  one. Each game pins to a stable color slot so the cabinet reads as
  three distinct blocks regardless of theme.
- Nonogram board: row hints right-anchored with a dark grid border
  and breathing room between numbers and the grid; cell-edge deadzone
  restored on swipes so glancing fingers no longer mark adjacent rows.
- Merge mode labels cleaned up ‚Äî "2048 ¬∑ Classic" / "Infinite ¬∑
  Endless" replaces the previous "Win mode (stop at 2048)" /
  "Infinite mode" copy.

## Internal changes
- `GameRecord.puzzleIdRaw: String?` (additive optional) lets the
  solved-puzzle gallery resolve completed puzzles back to
  `NonogramLibrary` entries. Not exported in v2 envelope (round-trip
  determinism preserved).
- `GameRoute` carries an associated mode/difficulty so the home
  drawer chips deep-link into a specific mode. Each game view's init
  accepts an optional initial mode that threads to its VM.

  (muted / bright) is derived from each preset's existing
  `PresetCategory`. `Theme.catalogueColor(_:)` resolves to the

## Fixes
- Nonogram swipe responsiveness ‚Äî restored the cell-edge deadzone
  that was dropped in 4d9b9da; fast diagonal swipes no longer leak
  into adjacent rows.
- Phase 9 VideoLocationPickerView UX ‚Äî replaced the iPhone-outline
  layout that overflowed on iPhone 17 Pro Max (and visually stacked
  small zones underneath large zones, suggesting the wrong mental
  model) with a vertical-stack picker: a Large/Small segmented size
  toggle at the top, then two stacked bands (Top/Bottom). Large mode
  renders each band as one tappable zone; Small mode places two
  corner buttons inside each band ‚Äî so the small video docks to a
  corner of one of the same two bands, never floats independently.
  Selected zone is bold + "Your video will go here" against a subtle
  accent fill; non-selected zones still read as tap targets. Flipping
  the size toggle preserves the user's Top/Bottom half. Fits without
  scrolling on iPhone 17 Pro Max and SE. 3 additive xcstring keys
  (`videoMode.locationPicker.size{Large,Small,A11yLabel}`); the six
  `videoMode.location.*` zone-label keys are unchanged. All 4 picker
  tests (`VideoLocationPickerViewTests`) GREEN.
