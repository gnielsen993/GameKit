`anchors.picker` is implicitly consumed via the single boolean
`let headerBarAtBottom = (anchors.headerBar == .bottomLeading || anchors.headerBar == .bottomTrailing)` —
in the small-zone cross-product, picker placement is always the inverse of HeaderBar placement, so a single boolean covers all 4 zones without independent picker branching.

| Zone | `anchors.headerBar` | `anchors.picker` | `headerBarAtBottom` | VStack render order | Clear edge |
|------|---------------------|------------------|---------------------|---------------------|------------|
| `.smallTopLeft` | `.bottomLeading` | `.bottomTrailing` | `true` | Board → HeaderBar → ModePill | top (away from top-PiP TL) |
| `.smallTopRight` | `.bottomTrailing` | `.bottomLeading` | `true` | Board → HeaderBar → ModePill | top (away from top-PiP TR) |

## Deviations from Plan

### [Rule 3 — Tooling] Transient parallel-execution commit-staging conflict, self-resolved

**Found during:** Final commit step (Task 2 done, all verification PASS).

**Issue:** This plan ran on the main working tree concurrently with the 12.1-03 (Merge) and 12.1-04 (Nonogram) sibling executors (`workflow.use_worktrees=false`). After my Mines edits were on disk and verified, the 12.1-04 Nonogram sibling executor's first `git commit` (commit `3d360c4`, since rewritten) swept up MY uncommitted Mines work along with the Nonogram changes. Before I could write this SUMMARY, the 12.1-04 sibling executor noticed its mis-staging and corrected the commit — `3d360c4` was replaced with `cf87f59` containing only the 2 Nonogram files. My Mines work was restored to the working tree, uncommitted, and I then created the proper atomic commit `78434bc` containing exactly the 2 Mines files this plan owns.

**Resolution:** Self-resolved by the sibling executor's correction. The final HEAD state is clean:
- `cf87f59 feat(12.1-04): wire smallZoneLayout in NonogramGameView` — 2 Nonogram files only.
- `a35f9ff docs(12.1-04): complete Nonogram smallZoneLayout wiring plan` — 12.1-04 SUMMARY.
- `78434bc feat(12.1-02): wire smallZoneLayout in MinesweeperGameView` — 2 Mines files only (this plan).

**Effect on plan goals:** Zero. Atomic commit honored. No cross-plan code residue. All acceptance criteria PASS.

**Forward note for the GSD framework:** When running parallel executors on a shared working tree (`workflow.use_worktrees=false`), there is a brief race window where one sibling's `git add`/`git commit` can absorb another sibling's uncommitted work. Mitigation in this run was automatic via sibling self-correction, but a more robust fix would be (a) enabling worktree mode for parallel executors, or (b) requiring each executor to stage by explicit file path list (which both 12.1-04 and 12.1-02 ultimately did).
