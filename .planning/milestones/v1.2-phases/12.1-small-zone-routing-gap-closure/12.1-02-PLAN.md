D-02 small-zone anchor values to consume:

| Zone | picker | headerBar | Semantic placement |
|------|--------|-----------|--------------------|
| `.smallTopLeft` | `.bottomTrailing` | `.bottomLeading` | ModePill bottom, HeaderBar bottom (both away from top-PiP) |
| `.smallTopRight` | `.bottomLeading` | `.bottomTrailing` | ModePill bottom, HeaderBar bottom |
| `.smallBottomLeft` | `.bottomTrailing` | `.topLeading` | ModePill ... see D-09 |
| `.smallBottomRight` | `.bottomLeading` | `.topTrailing` | ModePill ... see D-09 |

D-09 reading: when `anchors.picker` is `.bottomTrailing` / `.bottomLeading` AND we're on a Bottom L/R Small zone, the bottom-PiP overlay sits at the bottom — so the ModePill must render at the TOP of the board column instead. `anchors.headerBar` resolves to `.topLeading` / `.topTrailing` for Bot L/R, so HeaderBar stays at top. Result: HeaderBar (top) → ModePill (top, below HeaderBar) → Board fills below → bottom edge clear.

D-07 + D-08 reading: when `anchors.headerBar` is `.bottomLeading` / `.bottomTrailing` AND we're on a Top L/R Small zone, the top-PiP overlay sits at the top — so HeaderBar moves to BOTTOM of the board column. `anchors.picker` is also `.bottomTrailing` / `.bottomLeading` for Top L/R; per D-07/D-08, the visual top→bottom order is: Board → HeaderBar → ModePill (HeaderBar is the chips/state, ModePill is the picker; user reads chips above the picker).

So the cross-product collapses to two cases:
- **Top L/R zones (TL+TR)** — `anchors.headerBar` is a bottom anchor → render order is: Board → HeaderBar → ModePill (top of screen clear).
- **Bottom L/R zones (BL+BR)** — `anchors.headerBar` is a top anchor → render order is: HeaderBar → ModePill → Board (bottom of screen clear).

The cleanest in-code seam is a single boolean `let headerBarAtBottom = (anchors.headerBar == .bottomLeading || anchors.headerBar == .bottomTrailing)` and a parallel `let pickerAtTop = !headerBarAtBottom` (because ModePill is the inverse of HeaderBar in this cross-product). Implementer is free to add a tiny computed helper on `SlotAnchor` (`var isBottomAnchor: Bool`) as a code-clarity helper if file-size delta allows.
