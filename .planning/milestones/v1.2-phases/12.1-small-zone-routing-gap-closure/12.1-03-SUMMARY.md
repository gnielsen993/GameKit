## Decisions Made

1. **Shared sub-view extraction (`smallZoneBoard` / `smallZoneHeaderBar` / `smallZoneModePill`) vs. inline duplication.** The plan's CONTEXT D-MG-10 requires the `MergeBoardView` constructor + 3-line modifier chain to remain byte-identical across all renderings. The simplest expression is inline duplication (one copy per VStack ordering), but that pushed the file to 417 LOC — over the < 400 LOC acceptance target. Extraction to shared `@ViewBuilder private` properties enforces byte-identity at single source-of-truth AND brings LOC to 399. Trade-off: a tiny extra layer of indirection inside `smallZoneLayout`. `existingLayout` is **not** refactored to use these helpers — it stays inline so D-11 off-path byte-identity is mechanically preserved.

2. **`let _ = anchors.picker` line.** Acts as a self-documenting "we read this field" marker. The router (post-12.1-01) returns a value for `anchors.picker` that this code consumes through the `headerBarAtBottom` boolean derivation. Adding the explicit line is cheaper than a sentence-long comment and surfaces intent in `grep "anchors.picker"` output.

3. **Atomic single commit for Task 1 + Task 2.** Per CLAUDE.md §8.10 (one feature per commit) — `smallZoneLayout` creation + the body else-arm swap are one coherent feature. Splitting would have left an intermediate state where the new view exists but is unused.
