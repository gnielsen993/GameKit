    # Phase 13 — Win/Loss Banner + A11y Gating Manual Verification

    **Authored:** 2026-05-14 (Plan 13-05)
    **Filled in by:** Plan 13-05 (SC1 + SC2 + SC3 + SC4 + SC5 sweep)
    **Mirrors:** `.planning/phases/12-merge-nonogram-adoption/12-VIDEO-MANUAL-CHECK.md` per Phase 12.1 sign-off precedent.

    ## Purpose

    Single verifier-reads-end-to-end doc for Phase 13 SC1-5 per `.planning/ROADMAP.md` §"Phase 13: Win/Loss Banner + A11y Gating":

    - **SC1:** Win/loss surfaces in Video Mode use a non-board-covering banner/pill in all three games (Mines / Merge / Nonogram) — board remains fully visible, banner placement avoids the selected PiP zone in all 6 locations.
    - **SC2:** Primary CTA (Restart / Continue / New puzzle / Try again) is one-tap reachable from the moment banner appears — no expand-to-act, no tap-anywhere-on-banner pattern (08-BANNER-PLACEMENT D-11).
    - **SC3:** Haptics gating — `settingsStore.hapticsEnabled == false` silences banner haptic at the source (v1.0 05-03 D-10 FIRST-guard contract; unit-test-verified in `VideoModeBannerHapticsGateTests` from 13-01).
    - **SC4:** SFX gating — `settingsStore.sfxEnabled == false` silences banner SFX (default false per MINES-10); plays on `AVAudioSession.ambient` when enabled (does NOT duck user music — important since Video Mode implies external audio).
    - **SC5:** Animation + Reduce Motion gating — `accessibilityReduceMotion` dampens banner entrance + confetti + sweep to `.identity` / no-op per v1.0 05-06 D-04; legibility regression check passes on Classic + Loud (Dracula) for play / win / loss in all 3 games × 6 PiP zones.

    **Audit scale:** 36 rows = 3 games × 6 PiP zones × 2 outcomes (win + loss collapsed per zone-row for verifier ergonomics; auditor triggers win, marks, then triggers loss in the same zone before moving on). Mirrors the Phase 12.1 collapse pattern (12 rows → 12-VIDEO-MANUAL-CHECK matrix scale).

    **Worst-case rows:** Mines Hard × largeBottom (largest board, tightest banner squeeze) and Nonogram Medium-Hard × largeBottom (12pt cell floor + banner overlay) inherit the Phase 12-05 + 12.1 audit precedent and are the explicit acceptance rows. Other rows inherit by geometric interpolation.

    ## How to use

    For each matrix row:

    1. Launch the app on **iPhone 17 Pro Max simulator**.
    2. Settings → Video Mode → ON.
    3. Settings → Video location → select the row's `Zone`.
    4. Home → tap the row's `Game` tile.
    5. **Trigger win:** for the row's `Game`:
       - **Mines**: Easy → tap-cells-until-win. Verify banner appears with "You won!" + Restart CTA. Tap Restart → fresh board.
       - **Merge**: winMode → swipe-merge to 2048 (use sim debug if available; otherwise play through). Verify banner with "You reached 2048!" + Continue CTA. Tap Continue → infinite mode continues.
       - **Nonogram**: Tiny 5×5 → solve. Verify banner with "Solved" + puzzle title subtitle + "New puzzle" CTA. Tap New puzzle → fresh puzzle.
    6. **Trigger loss:** for the row's `Game`:
       - **Mines**: tap a mine. Verify banner with "Bad luck" + Restart. Tap Restart → fresh board.
       - **Merge**: play to gameOver (no moves). Verify banner with "Game over" + Restart. Tap Restart → fresh board.
       - **Nonogram**: trigger 3 wrong fills. Verify banner with "Out of lives" + "Try again". Tap Try again → same puzzle reset.
    7. **Verify banner geometry:** banner sits at the edge OPPOSITE the selected PiP zone (per 08-BANNER-PLACEMENT D-09 table). Board remains fully visible behind banner.
    8. **Verify off-restore:** Settings → Video Mode → OFF. Replay the same win/loss in the same game; verify the BASELINE full-screen `EndStateCard` appears (NOT the banner). This is the VIDEO-13 spot-check.
    9. Mark Pass/Fail per row. Add notes for any deviation.

    **Preset audit (SC5 inner loop):** repeat steps 1-7 with Settings → Theme → Dracula (the Loud preset per CLAUDE.md §8.12). At minimum verify worst-case rows (Mines × largeBottom, Nonogram Hard × largeBottom) on Dracula.

    **A11y gating spot-checks (one-shot):**
    - **SC3 Haptics gate:** Settings → Haptics → OFF. Trigger one win in Mines on largeBottom. Confirm NO haptic fires. Toggle back ON; trigger another win; confirm haptic fires.
    - **SC4 SFX gate:** Settings → SFX → ON (default is OFF). Trigger one win. Confirm SFX plays. Start Music app playback in background; trigger another win. Confirm music does NOT duck (banner SFX is `AVAudioSession.ambient`). Toggle SFX OFF; trigger win; confirm silence.
    - **SC5 Reduce Motion gate:** iOS Settings → Accessibility → Motion → Reduce Motion → ON. Re-launch app. Trigger one win in Mines on largeBottom. Confirm banner appears WITHOUT fade transition (instant). Confirm confetti + win-sweep wash collapse to no-op.
