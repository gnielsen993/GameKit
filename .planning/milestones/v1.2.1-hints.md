# Milestone Sketch: v1.2.1 — Hints

**Status:** Idea — not committed  
**Type:** Patch (small, ships fast)  
**Created:** 2026-05-22

---

## What ships

A hint system added to the four games where hints have a clear, unambiguous meaning. One hint action per game. A fixed allowance per puzzle — no coins, no earning, just a small pool that resets each new game.

---

## Per-game scope

| Game | Hint action | Notes |
|------|-------------|-------|
| Minesweeper | Reveal one guaranteed-safe unrevealed cell | Use constraint solving or a seeded safe-cell list; never reveals a mine |
| Nonogram | Fill one correct cell in the grid | Pick the cell that gives the most new information (longest unsatisfied run) |
| Sudoku | Place one correct digit in an empty cell | Straightforward — solution is known at puzzle load |
| Merge | — | No natural hint; skip this game |

---

## Allowance design

- **3 hints per puzzle** across all games — simple, matches user expectation from crossword / Sudoku apps
- Allowance resets on new game / new puzzle; does not persist across sessions
- No hint bank, no earned hints, no store purchase — consistent with zero-monetization stance
- If hints are exhausted: button dims and shows "0" — no upsell, no modal

---

## UI

- Hint button lives in the toolbar / control row alongside restart — exact placement follows each game's existing chrome (see DESIGN.md §3, §12)
- Icon: lightbulb (SF Symbol `lightbulb` or `lightbulb.fill`) — universally understood
- Counter badge on the button showing remaining hints (3 → 2 → 1 → 0)
- Tapping with 0 remaining: brief haptic rejection, no other feedback
- Revealed hint cell gets a brief pulse animation (same glow token as Sudoku completion glow) to draw the eye

---

## Open questions before committing

1. **Minesweeper hint algorithm** — pure random safe-cell pick is fine for Easy; for Hard, prefer a cell that makes the most unrevealed region safe. Worth a quick spike.
2. **Nonogram hint picking** — random correct cell vs. "most useful" cell. Random is simpler and probably fine for v1.2.1.
3. **Does the hint count appear in stats?** Likely no — adds complexity for minimal value.
4. **Accessibility** — hint button needs a VoiceOver label that reads remaining count ("2 hints remaining").

---

## Complexity

Small. Each game's hint action is a pure function on the existing model — no new persistence, no new game state shape. The UI addition is a single button + counter per game.

Estimate: 2–3 implementation sessions, one for each of Minesweeper / Nonogram / Sudoku, plus a shared HintButton component.
