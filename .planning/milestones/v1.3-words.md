# Milestone Sketch: v1.3 — Word Games

**Status:** Idea — not committed  
**Type:** Minor (new game milestone, two games)

## What ships

Two word games added to the suite. One easy entry point (Wordle-style), one harder logic game (Mini Crossword). Both use the same underlying dictionary service and share a word-list asset.

---

## Game 1: Word Guess (Wordle-style)

**Difficulty:** Easy — accessible, broad audience, familiar format  
**Core loop:** Guess a hidden 5-letter word in 6 tries. Each guess scores each letter: correct position (green), wrong position (yellow), not in word (grey).

### Mechanics
- 5 letters, 6 guesses — the established format; don't deviate
- Hard mode: each guess must use confirmed letters in confirmed positions
- Daily puzzle (same word for all players on a given date, derived from a seeded list) + free-play mode (random word each time)
- Keyboard on-screen with letter state coloring — reuses the grid color semantics

### Design questions
- **Color mapping to DesignKit tokens:** correct = `theme.colors.success` (or accent?), wrong position = `theme.colors.warning`, absent = surface/muted. Need to verify these tokens exist and are legible on all presets — especially Dracula where green success may clash. Resolve before implementation.
- **Daily word cadence:** Pre-baked list of ~365 curated words shipped in the bundle, indexed by calendar day. No network required. Overflow to free-play after year one.
- **Word list licensing:** Must use a public-domain or permissively licensed word list (e.g. SCOWL, Wordle's original published list). Confirm before shipping.
- **Sharing:** "Copy result" emoji grid (⬜🟨🟩) is expected by users of this genre. Include it. No share sheet required — clipboard only.

## Game 2: Mini Crossword

**Difficulty:** Hard — logical, deliberate, satisfying  
**Core loop:** Fill a 5×5 grid using Across and Clue hints. All answers intersect. No blank squares outside the grid.

### Mechanics
- 5×5 grid — same scale as NYT Mini; tractable to generate and to play on a phone
- Clues are short (3–8 words), factual or wordplay — no trivia requiring specialist knowledge
- Tap a cell → highlights the word it belongs to, shows the clue → type letters
- Pencil mode (uncertain letter, shown lighter) matches Sudoku's pencil-note convention
- Backspace clears; tapping a completed word cycles to the next unfilled word

### Generation vs. curation
- **Recommend: curated puzzles, not generated.** A good crossword clue is editorial work. Generated grids with auto-clues read as low-quality. Ship 100–200 hand-curated 5×5 puzzles at launch, stored as a JSON bundle. This also sidesteps the constraint-solver complexity.
- Completed puzzles tracked so users don't repeat — same pattern as Sudoku.

### Design questions
- **Grid cell size on small devices:** 5×5 at full width gives ~60pt cells on an iPhone SE — comfortable. Verify.
- **Clue display placement:** Below the grid, not overlaid. Single clue shown at a time (the active word).
- **Classic preset:** Standard white cells, black blocked squares. No felt. Crossword grids look wrong on felt.

## Shared infrastructure

Both games need:
- `WordDictionaryService` — validates guesses against a word list; pure Swift, no SwiftUI
- A curated word-list asset (separate from the full dictionary used for validation)
- `WordGameStats` model — games played, win rate, current streak, best streak (Word Guess) / puzzles solved (Mini Crossword)

`WordDictionaryService` is a good candidate for its own Swift file in `Core/` since both games use it. If it stays under 100 lines, put it in the game folder and promote later.

---

## Complexity

Medium-high. The games individually are tractable; the complexity is in word-list sourcing, curation of Mini Crossword puzzles, and the keyboard/grid input components which are new to the codebase.

Word Guess is probably 4–5 implementation sessions. Mini Crossword is 5–7 (input model + grid renderer + clue panel + curation pipeline + stats). A combined milestone makes sense since they share the dictionary asset and the word-genre positioning in the app.
