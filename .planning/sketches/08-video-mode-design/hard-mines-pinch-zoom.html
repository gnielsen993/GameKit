Variant 3: Pinch-zoom — reuse A11Y-05

The Hard 16x30 board reuses the existing A11Y-05 pinch-zoom from Plan 06.1-03. The only new behavior in Video Mode is an optional one-shot auto-fit animation on Video Mode entry — animate zoomScale from 1.0 to whatever scale fits the 16x30 board between the PiP band and the compact row. After that, the user manages the zoom with the existing pinch system.

A11Y-05 / 06.1-03 deconfliction (LOWEST RISK — zero new gestures): this variant reuses the existing MagnifyGesture + clampZoomScale + .scaleEffect layer verbatim. The only added behavior is a one-shot withAnimation { zoomScale = autoFitScale } when Video Mode toggles on (and Hard is the current difficulty). No gesture composition change. No new state. Cell-level .exclusively(before:) stays byte-identical.

Behavior

  • Cell size: unchanged at the source level — 18pt floor preserved. Visual size on screen is 18pt × zoomScale.
  • Auto-fit: on Video Mode entry, animate zoomScale from current value to fitScale = (availableHeight - 2*padding) / (rows * (18 + spacing)), clamped to the existing 0.8–2.0 range.
  • If auto-fit can't reach a satisfying fit (i.e. fitScale < 0.8), zoomScale stops at 0.8 floor and ScrollView fallback engages (06.1-03 D-16 — already shipping).
  • User control: pinch to expand back to normal size, then scroll to play; or accept the smaller scaled view and play without scrolling.
  • Reduce Motion: auto-fit animation is gated behind accessibilityReduceMotion — falls back to .identity (instant snap to fitScale).

Pros

  • Lowest code surface — only an auto-fit trigger on Video Mode entry.
  • Zero new gesture surface; reuses A11Y-05 verbatim.
  • User already knows how to use pinch from non-Video-Mode play (no new affordance to teach).
  • Easiest to ship; smallest Phase 11 plan footprint.

Cons

  • Discoverability: without auto-fit, Hard + Large PiP renders unplayable at first glance — user must know pinch works.
  • Auto-fit clamps at 0.8x. If fitScale < 0.8, fallback ScrollView engages and user sees a partial board until they pinch or scroll — a worse initial state than Variant 1 (smaller cells, full board visible).
  • Animated auto-fit on entry may collide with the cell reveal-cascade animation from Phase 5 if the user re-enters Video Mode mid-game.
  • Triggers Phase 11 research-flag per ROADMAP §v1.2 Research Flags (composition with cell gestures during the auto-fit animation needs verification).
