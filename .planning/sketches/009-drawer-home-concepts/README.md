# Sketch 009: Drawer Home Concepts

## Design Question
The name "GameDrawer" isn't landing — the current icon grid feels like a generic launcher.
Which interaction pattern best embodies the drawer metaphor while staying fast and discoverable?

## How to View
```
open .planning/sketches/009-drawer-home-concepts/index.html
```

## Variants

- **A: Accordion Drawer Rows** — Full-width game rows, each is a "drawer" with a handle mark on the right. Tap to pull it open in-place; modes/difficulty slide out below. Left accent bar lights up in the game's color. All games always visible.

- **B: Bottom Sheet Drawer** — Home screen shows app identity (wordmark + recent games). The actual games live in a bottom sheet that peeks 178px from the edge. Tap the grip to slide it fully open. Most literal drawer metaphor.

- **C: Pull-Tab Grid** — Preserves the current 3-col icon grid. Each tile gets a small horizontal "pull handle" pip below it. Tapping scales the tile up, the pip lights up in accent color, and a detail panel slides open below that row — pushing everything down like a drawer being pulled out.

## What to Look For
- Does A feel too list-like / Settings-y, or does it feel premium?
- Does B's home screen have enough identity to justify hiding the games behind a sheet?
- Does C's pull-handle communicate "drawer" without explaining it, or does it read as a loading indicator?
- Which one makes you want to open games vs. which feels like work?
