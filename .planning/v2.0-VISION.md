## 1. Thesis

**One line:** A single **rebrand relaunch** that turns a technically-strong
10-game engine into a *finished, kind, inviting Watch Mode hub* ‚Äî polished
product surface, a curated roster expansion, and an honest paywall, all landing
under one 2.0 banner.

The platform is ~80% built. 2.0 is not a code rebuild ‚Äî it is a **product-surface,
polish, and identity push** that makes what already exists *feel* like something
a casual player wants to open, and re-presents the app as one coherent product
rather than "10 games I made." The bands below are the content the 2.0 identity
shift draws from ‚Äî though which of them ship *as* 2.0 vs. early as a 1.x is a
ship-time call (see ¬ß1.1).

The trap to avoid (in the builder's own words): shipping "10 games I made."
The target feeling: **a polished Watch Mode game hub with a growing library.**

---

## 1.1 Versioning philosophy ‚Äî the number is earned at ship time

Version numbers are **not** pre-assigned to the bands in ¬ß5. They're decided by
the *magnitude of what actually ships together*:

- **2.0 is reserved for the identity shifter.** Whatever bundle of work
  genuinely re-presents the app (e.g. full home rebrand + daily-challenge focus
  shipping as one drop) claims the 2.0 label. 2.0 must *feel* like a new app,
  not a routine update.
- **Anything shipped before the identity shift is a 1.x point release.** If the
  Assist layer (Band C) is ready and we want it out the door before the rebrand
  lands, it ships as **1.6** ‚Äî a normal point release ‚Äî and 2.0 stays reserved
  for the identity moment that comes later.
- **Anything shipped after 2.0 is a 2.x.** Same Assist work, if it lands *after*
  the identity shift, is **2.1** instead of 1.6. The work is identical; the
  label depends only on where it falls relative to the identity shift.

So the bands are a backlog we pull from. Which ones get grouped into the 2.0
drop vs. shipped early as 1.x vs. deferred to 2.x is a **ship-time call based on
where we are**, not fixed here. The one rule: **don't spend "2.0" on a small
idea.** If a smaller thing turns out to be the identity shifter, *that* becomes
2.0.

## 2. Where we are (current state, verified 2026-07-07)

**Games (10):** Minesweeper ¬∑ Merge ¬∑ Nonogram ¬∑ Sudoku ¬∑ Klondike ¬∑
FreeCell ¬∑ FiveLetter ¬∑ WordGrid ¬∑ Stack ¬∑ Snake.

**Platform ‚Äî already built and shared:**
- **Video Mode** ‚Äî real app-wide layout engine (`VideoModeStore`, 6 PiP
  zones, `VideoModeSlotRouter`). 8/10 games adopt it; Snake + Klondike exempt.
  *This is the moat and it exists.*
- **Save-state** ‚Äî per-game persistence for all 8 non-arcade games
  (`*SaveState.swift`). *Buried in-game; not surfaced.*
- **Arcade primitive** ‚Äî reusable endless-loop substrate (`ArcadeGameState`,
  `ArcadeLoopDriver`, v1.5). Stack + Snake ride it.
- Onboarding (`IntroFlowView`), settings, haptics, SFX, DesignKit themes,
  iCloud/SIWA auth ‚Äî all present.

**Platform ‚Äî absent:** Daily Challenge system ¬∑ cross-game streak (Stack-only
today) ¬∑ Game Center ¬∑ Monetization.

**Product surface ‚Äî the weak layer:**
- Home is a flat 3-column tool grid with a drawer-expand interaction. Reads
  as *internal / techy*, not consumer-inviting.
- Onboarding is a setup wizard (Theme ‚Üí Features ‚Üí Sign In); it never
  *pitches* the app.
- No surfaced "Continue Playing" despite save-state existing everywhere.
- Default palette is Chrome Diner (a tasteful restomod, not the friendliest
  first impression for a casual user).
- Games are win/lose binary ‚Äî no dignified exit from a stuck moment.

## 3. The five consumer truths driving 2.0

These came from looking at the app *as a player, not a builder*:

1. **Onboarding doesn't sell the app.** It configures; it never explains why
   to be excited.
2. **Home isn't inviting.** It's a tool grid. Consumer apps guide you to a
   first action; this dumps you in a menu.
3. **No reason to come back.** No daily hook.
4. **No kindness in a stuck moment (the hidden one).** Thinking games with no
   hint/undo/help make players feel like *winners or losers* ‚Äî never "I did
   it with one hint." This is what separates a prototype-feel from a
   polished-feel.
5. **Default palette isn't the friendliest.** The connoisseur Classic skin
   shouldn't be the casual first impression.

---

## 4. Principles / guardrails (these shape every milestone)

- **Monetize value, not compulsion.** No dopamine loops, no coins, no energy,
  no manufactured friction. The honest model: charge a small, flat price for a
  genuinely useful feature (Video Mode), keep the entire base game free forever.
  Same price whether they play 2 minutes or 2 hours.
- **Assists are free and generous ‚Äî never a currency, never a paywall lever.**
  The moment hints cost money, we've built the F2P thing we hate.
- **Assists are opt-in.** Purists can turn them off; the "calm premium" core
  value is preserved.
- **Polish before quantity.** No new game ships to the main library until it
  passes the quality bar (¬ß7). Experiments hide in a Labs/Beta shelf.
- **Constitution discipline.** Monetization and any second network surface
  (Game Center) contradict CLAUDE.md ¬ß1 as written. They are **DECISION GATES**
  requiring an explicit ¬ß1 amendment + ADR *before* build ‚Äî not assumed scope.

## 5. v2.0 build order (phase bands within the one milestone)

A backlog of work bands, in recommended build order (consumer-grade *before*
monetized ‚Äî you can't sell a feature the app never showed off). Version labels
are assigned at ship time per ¬ß1.1: the identity-shifting bundle becomes 2.0;
anything shipped earlier is a 1.x point release; anything later is a 2.x. These
are build phases (continuing GSD numbering from Phase 18), not fixed versions.

### Band A ‚Äî "First Impression" (product-surface polish)
**Goal:** A new user's first 30 seconds feel like a consumer app, not a dev tool.
- **Onboarding rewrite** ‚Äî lead with the *pitch* (calm games while you watch ‚Üí
  show Video Mode in one beat) *before* asking for theme/sign-in.
- **Home restructure** ‚Äî sectioned, scrollable, inviting:
  `Continue Playing` ‚Üí `Daily` (placeholder until v1.7) ‚Üí `Featured` (3‚Äì5) ‚Üí
  `Best for Watch Mode` ‚Üí `Classics` ‚Üí `All Games`. Resolve the tension with
  the current drawer-expand interaction via a sketch first.
- **Continue Playing** ‚Äî surface the existing per-game save-states as home's
  top section. Mostly plumbing; data already persisted.
- **Warmer default palette** ‚Äî ship a softer, friendlier preset as the
  first-run default; keep Classic one tap away. *(DesignKit default-preset
  change ‚Äî cross-repo implication, update ¬ß0.1.)*
- **Done bar:** first-run walkthrough + home reviewed on Classic + one
  Loud/Moody preset; a non-technical tester can name what to play first.

### Band B ‚Äî "Daily Hook" (retention platform gap)
**Goal:** A reason to come back tomorrow.
- **Shared Daily engine** ‚Äî seeded-by-date generation + `DailyChallengeStore`.
- **Generalized cross-game streak** ‚Äî promote Stack-only streak to a shared
  system; retrofit FiveLetter's ad-hoc daily onto the real engine.
- **Daily section on home** goes live.
- **Done bar:** daily puzzle is deterministic per date, streak survives
  relaunch + iCloud round-trip.

### Band C ‚Äî "Be Kind" (the Assist layer)
**Goal:** No player feels like a loser in a stuck moment.
- **Shared, opt-in Assist layer** ‚Äî per-game but consistently framed: hint
  (highlight a valid move), undo, extra FreeCell free cell, reveal-a-cell
  (Sudoku), reveal-a-line (Nonogram).
- **Dignified accomplishment** ‚Äî record the win, note the help gently
  ("Solved ‚Äî with 1 hint"). Never blocked, never punished.
- **Free forever** ‚Äî assists are *not* monetized (guardrail ¬ß4).
- **Done bar:** every thinking game has ‚â•1 assist; assists honor the
  haptics/animation settings; purist mode (assists off) works cleanly.

### Band D ‚Äî "Roster" (curated casual expansion)
**Goal:** Prove the niche + add market pull, on the existing engine.
- **Color Sort first** ‚Äî the purest Watch Mode game (fully turn-based, zero
  time pressure, glanceable). Niche proof-point. Rides the video-mode engine;
  no arcade loop needed.
- **Block Puzzle second** ‚Äî strongest general-market fit; the flagship.
- Both pass the ¬ß7 quality bar before entering the main library.
- **Done bar:** both games ¬ß12.5-checklist clean, Video Mode adopted,
  save-state + empty/win states + assists where relevant.

### Band E ‚Äî "Rebrand & Premium" (identity + honest paywall ‚Äî the launch moment)
**Goal:** Re-present the app as a Watch Mode hub and charge for value. This is
the band that makes it *2.0* ‚Äî the identity refresh + paywall land together as
the public relaunch.
- **DECISION GATE: amend CLAUDE.md ¬ß1** to permit a single, non-aggressive
  subscription. Write an ADR. *(Blocks all build below.)*
- **Paid tier = Video Mode** (the must-have) **+ premium Themes** (the
  nice-to-have upsell). ~$1‚Äì2/month. Everything else free forever.
- **Paywall placement** ‚Äî Video Mode is now the storefront the v1.6 onboarding
  + home have been showing off. Elegant, non-pushy; base game never gated.
- **Identity refresh (the rebrand)** ‚Äî this is where 2.0 earns its number: the
  app re-presents itself as a Watch Mode hub. Refresh positioning, App Store
  presence, and any ¬ß0.1 brand facts that move; update `~/Desktop/GameKitWebsite`
  per ¬ß8.15 wrap-up. Everything from Bands A‚ÄìD is what makes this credible.
- **App Store 2.0** ‚Äî reposition marketing around the niche: "games built for
  divided attention," market 3‚Äì5 games not 10, "growing library of Watch Mode
  games."
- **Done bar:** subscription flows (purchase / restore / lapse ‚Üí graceful
  Video-Mode-off) tested; free base game fully playable signed-out and unpaid.

## 7. Quality bar (main-library gate ‚Äî enforced from v1.6 on)

Before a game is visible in the main library it must:
- work in Video Mode (or be a deliberate, documented exemption like Snake),
- have save/resume,
- have clean empty / error / win states,
- have haptics or visual feedback,
- have consistent theme styling across ‚â•4 presets,
- be understandable within 5 seconds,
- not look like a prototype,
- have at least one reason to replay,
- (v1.8+) offer an assist where it's a thinking game.

Fails any ‚Üí Labs, or don't ship.

## 9. Why this order (within the one milestone)

Polish (A) ‚Üí retention (B) ‚Üí kindness (C) ‚Üí roster (D) ‚Üí rebrand + money (E).

The paywall and rebrand are Band E on purpose: Bands A‚ÄìD build the *thing worth
paying for* and the *storefront that sells it*. Monetizing before the app feels
consumer-grade would charge for an unfinished feeling. By the time Band E lands,
onboarding and home have spent four bands showing off Video Mode ‚Äî so asking
$1‚Äì2/month for it is the honest close, not a pushy ask. And because it all ships
as one 2.0 relaunch, the version jump reads as a genuine identity change, not a
routine update.

(¬ß1.1) ‚Äî the phase numbers don't care; only the marketing version does.
