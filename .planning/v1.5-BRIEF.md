# v1.5 — Endless Arcade Primitive

**Status:** brief (pre-milestone) · **Created:** 2026-06-25
**One-liner:** Expand the suite into a new *category* — real-time endless
arcade — by building one reusable game-loop substrate and shipping two
**calm** endless games on top of it.

> Formalize into phases later with `/gsd-new-milestone v1.5`. This file is
> the agreed scope going in.

---

## Why this milestone

Every shipped game (Minesweeper · Merge · Nonogram · Five Letter · Word
Grid · FreeCell · Sudoku) is **turn-based, deterministic logic**. v1.5 is
the deliberate breadth move: prove a brand-new interaction primitive —
*continuous input + frame loop + score-until-death* — once, then reuse it.

**Brand guard:** calm-only for this release. No rage/twitch games yet (see
`UPCOMING-GAMES.md` for where Flappy-style etc. go). Calm endless =
meditative, geometric, theme-token-perfect, "one more try" without stress.

---

## Scope — what ships in v1.5

### 1. Shared real-time substrate (the actual point of the milestone)
Lives in `Core/` (cross-game). Built once, reused by both games:

- **Loop driver** — `TimelineView(.animation)` (or `CADisplayLink`) at
  device refresh rate, feeding a fixed/variable `dt` to a pure engine.
- **Pure tick engine contract** — keep the spine: the engine is
  Foundation-only and deterministic. Shape:
  `mutating func step(dt: Double, input: Input) -> Frame`. Physics and
  collision are deterministic given (seed, input stream) → unit-testable
  exactly like the logic engines.
- **Run lifecycle shell** — idle → running → game-over → restart, with a
  tap-to-start affordance and a game-over banner (reuse the v1.2
  `VideoModeBanner` pattern where it fits).
- **High-score persistence** — extend the existing SwiftData stats layer
  (`GameRecord` / `BestTime`) with a high-score concept rather than a new
  store. Survives force-quit per §6 done bar.

### 2. Stack (flagship calm endless)
Tap to drop a sliding block onto the tower; overhang is shaved off; tower
narrows until you miss. Geometric, theme-driven, zen. Pure engine:
`StackEngine.drop(at:) -> (placed, overhang, newWidth, gameOver)`.

### 3. Snake (classic calm endless)
Grid-based, universally understood, reuses board-rendering instincts.
Swipe/tap-to-turn. Pure engine: `SnakeEngine.step(dir:) -> Frame`
(grow / self-collision / wall rule as a config flag).

## Open decisions to resolve in planning

- **Video Mode:** real-time continuous-input games are a poor fit for
  PiP reflow (can't pause-and-reflow mid-run). Likely **Video-Mode-exempt
  for v1.5**, or a minimal "pause when banner shows" treatment. Decide in
  discuss-phase; don't assume the v1.2 reflow contract applies.
- **Engine purity vs. frame timing:** lock fixed-timestep vs. variable
  `dt`. Fixed-timestep keeps tests trivially deterministic — preferred.
- **Difficulty / ramp:** endless games ramp via speed, not difficulty
  presets. Define the ramp curve per game (no Easy/Medium/Hard rows).
- **Stats shape:** high score + runs played + (maybe) longest run. Confirm
  what the Stats screen shows for a score-based game vs. a win/loss game.
- **Reduced Motion:** these are motion-heavy. Plan a Reduce Motion path
  (slower/no parallax, no screen shake) per §1 accessibility rules.

## Out of scope (explicitly)
- Twitch/reflex games (Flappy-style, rhythm-tap) → `UPCOMING-GAMES.md`.
- Chess puzzles (own milestone — needs move engine + puzzle DB).
- Word/Solitaire/Sudoku **depth** push (later milestone).
