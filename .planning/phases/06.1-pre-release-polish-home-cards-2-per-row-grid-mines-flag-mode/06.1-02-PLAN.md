<objective>
Add a Reveal/Flag interaction-mode toggle to Minesweeper. Two modes: `.reveal` (default — tap reveals, long-press flags — current MINES-02 contract) and `.flag` (tap toggles flag, long-press reveals — inverted). Long-press always provides a quick-action escape from the current mode. The toggle UI is a floating button (FAB) below the board, color-coded and icon-swapped per mode, with light haptic on toggle. Mode resets to `.reveal` on every `restart()`. The 0.25s long-press threshold and `.exclusively(before:)` gesture composition are preserved VERBATIM — only the *meaning* of the gesture branches on `interactionMode`.

Purpose: Address user-reported pain point — long-press-to-flag is slow/awkward when flagging many cells in a row. A mode toggle gives the user a one-tap-to-flag affordance without breaking the muscle memory of long-press-anywhere as a quick-action escape. MVP-grade ergonomic improvement on shipped P3/P5 surface.

Output: MINES-12 satisfied. Atomic commit per CLAUDE.md §8.10. Zero regression on ROADMAP P3 SC1 (50-tap zero-misfire) or P5 cell-level haptic contract.
</objective>
