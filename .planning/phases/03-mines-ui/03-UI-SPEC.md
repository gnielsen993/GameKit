## Spacing Scale

Declared values (DesignKit `SpacingTokens` ŌĆö already on disk at `../DesignKit/Sources/DesignKit/Layout/SpacingTokens.swift`):

| Token | Value | Usage in Phase 3 |
|-------|-------|------------------|
| `theme.spacing.xs` | 4pt | **Cell-to-cell grid gap** (LazyVGrid `spacing: theme.spacing.xs`); flag/mine glyph inner padding inside cell tile |
| `theme.spacing.s` | 8pt | Header bar internal padding (counter + timer chips); end-state card outcome-line vertical rhythm; toolbar item gaps |
| `theme.spacing.m` | 12pt | Header bar horizontal screen-edge inset; gap between header and board; gap between board and toolbar (when present) |
| `theme.spacing.l` | 16pt | `DKCard` outer padding (already baked into `DKCard` ŌĆö D-04 confirms); end-state card title-to-body gap; board container horizontal padding on Easy/Medium |
| `theme.spacing.xl` | 24pt | End-state card vertical stack rhythm between major sections (title block ŌåÆ metadata ŌåÆ action stack) |
| `theme.spacing.xxl` | 32pt | Top inset between safe area and HeaderBar; bottom inset above home indicator on game scene |

Exceptions:
- **Cell tile dimensions** are *not* spacing tokens ŌĆö they are intrinsic component dimensions chosen for touch-target safety. **44├Ś44pt minimum** (Apple HIG; matches `DKButton`'s `minHeight: 44`) on Easy 9├Ś9. **Hard 16├Ś30 falls back to 36├Ś36pt with horizontal `ScrollView`** per CONTEXT Discretion (b) "horizontal scroll on Hard." Medium 16├Ś16 stays at **40├Ś40pt fixed** (fits 16 ├Ś 40 + 15 ├Ś 4 = 700pt ŌĆö does NOT fit 320pt iPhone SE width, so Medium also scrolls horizontally on SE; it fits comfortably on iPhone 13+ at 390pt+).
- **Cell tile values are derived constants in `MinesweeperBoardView`**, NOT new spacing tokens. They live as `private let cellSize: CGFloat` because they are component-intrinsic, not generic spacing rhythm. CLAUDE.md ┬¦1 forbids hardcoded `padding(<int>)` and `cornerRadius:` integers; bare constants used as `.frame(width: cellSize, height: cellSize)` are not flagged by the FOUND-07 pre-commit hook (which scans `padding(`, `cornerRadius:`, `Color(`).

**Hard cap:** Zero raw `.padding(8)` / `.padding(.horizontal, 16)` integer literals in `Games/Minesweeper/`. Every padding call site reads `theme.spacing.{token}`.

| Role | DesignKit token | Used For | Notes |
|------|-----------------|----------|-------|
| Display | `theme.typography.titleLarge` (32pt, rounded, bold) | End-state card outcome title ŌĆö "You won!" / "Bad luck" | Outcome title is the only Display-scale text in P3 |
| Heading | `theme.typography.title` (22pt, semibold) | Toolbar Menu trigger label ("Easy" / "Medium" / "Hard"); end-state metadata line ("2:14") | Two heading slots in P3 |
| Body | `theme.typography.body` (17pt, regular) | Loss-only context line ("3 mines hit / 47 safe cells left"); Restart / Change difficulty button labels (passed to `DKButton` which applies `theme.typography.headline` internally ŌĆö see DKButton.swift line 32) | Default body for prose surfaces |
| Mono / Counter | `theme.typography.monoNumber` (17pt, monospaced) | **Mine counter** ("`042`") and **timer** ("`02:14`") in HeaderBar | Monospaced = numbers do not jitter on every second tick (T 0:09 ŌåÆ 0:10 must not shift the layout right of it). Required for SC2. |
| Cell adjacency | `.system(size: cellSize ├Ś 0.55, weight: .bold, design: .rounded)` | The "1"ŌĆō"8" digit inside a revealed numbered cell | **Exception** ŌĆö adjacency text scales with cell size, not Dynamic Type. Color is `theme.colors.gameNumber(adjacent)`. |

**Roles count:** 4 declared roles + 1 grid carve-out = within "3-4 sizes, 2 weights" budget once you collapse `.regular` (body) and `.semibold` (title/headline) into "2 weights" + the carve-out bold for cell digits.

**No `.font(.title2)` / `.font(.system(size: 18))` literals anywhere in `Games/Minesweeper/`.** Every `.font()` call site reads `theme.typography.{token}` except the documented cell-adjacency carve-out.

## Color

DesignKit operates on **semantic tokens** rather than fixed hex values ŌĆö every preset (34 across 6 categories) ships its own resolved palette through `Theme.resolve(preset:scheme:)`. The 60/30/10 split is enforced by **token role**, not hex:

| Role | Token | Usage in Phase 3 |
|------|-------|------------------|
| Dominant (60%) | `theme.colors.background` | Game scene root background; mine-board "gutter" between cells when cell gap (`spacing.xs`) is visible |
| Secondary (30%) | `theme.colors.surface` (revealed-cell fill, header chips, end-state card body via `DKCard`); `theme.colors.surfaceElevated` (unrevealed-cell fill ŌĆö slightly raised vs revealed to read as "tappable") | The revealed-vs-unrevealed visual hierarchy is conveyed by `surface` vs `surfaceElevated` ŌĆö **not by tinting**. Both tokens already exist in `ThemeColors` and resolve correctly under all 6 preset categories. |
| Accent (10%) | `theme.colors.accentPrimary` | **Reserved exclusively for** (1) the **Restart** primary action (`DKButton(.primary)` already uses `accentPrimary` background ŌĆö DKButton.swift line 56); (2) the toolbar Menu trigger glyph tint when the Menu is open. Nothing else in `Games/Minesweeper/` reads `accentPrimary`. |
| Destructive (semantic) | `theme.colors.danger` | (a) End-state card title color **on loss only**; (b) the mine-cell that triggered loss ŌĆö its background fill turns to `danger` per D-17 step 1; (c) the X overlay on incorrectly-flagged cells per D-17 step 3; (d) destructive confirmation alert action ("Abandon" in the mid-game-switch alert per D-10) |
| Success (semantic) | `theme.colors.success` | End-state card title color **on win only** (per SC4 + D-04). Nothing else in Mines reads `success` in P3. |
| Numbered cells (NEW) | `theme.colors.gameNumber(_ n: Int)` (1ŌĆō8) | Adjacency-number text color on revealed numbered cells. Per-preset 8-color array (D-14). Default Classic palette: traditional Minesweeper (1=blue, 2=green, 3=red, 4=dark blue, 5=maroon, 6=cyan, 7=black, 8=grey). Loud presets ship neon variants tuned to the preset's accent. |
| Text primary | `theme.colors.textPrimary` | All non-accent text: end-state body copy, header counter/timer digits, mine-glyph fill (mine = `circle.fill` tinted `textPrimary`), toolbar menu labels |
| Text secondary | `theme.colors.textSecondary` | A11y-only ŌĆö never visible directly. Used for VoiceOver-rendered hint text only if needed in P5. P3 unused. |
| Border | `theme.colors.border` | `DKCard` overlay stroke (already baked in ŌĆö DKCard.swift line 19); cell tile 1pt stroke distinguishing revealed-empty vs revealed-numbered cells under low-contrast presets (Cream, Sage) |

**Accent reserved-for list (explicit):**
1. Restart `DKButton(.primary)` background
2. Toolbar Menu open-state glyph tint

**Two items.** Nothing else. The Mines toolbar's Restart glyph (`arrow.counterclockwise`) when *unpressed* renders on `theme.colors.textPrimary` so the accent is reserved for the moment the user is committed to acting (the filled-button state inside `DKButton(.primary)`).

## Motion

P3 ships **functional defaults only** ŌĆö Phase 5 (MINES-08) owns the polished cascade (reveal stagger, win sweep, loss shake). P3 must lay out V/VM contracts so P5 can layer `MinesweeperPhase` enum-driven animations without changing the V/VM surface (D-18).

| Transition | Duration token | Animation | Justification |
|------------|----------------|-----------|---------------|
| Cell tap ŌåÆ reveal flip | `theme.motion.fast` (0.18s) | `.easeInOut` | Tap feedback must be near-instantaneous; longer durations feel laggy |
| Flag toggle | `theme.motion.fast` (0.18s) | `.easeInOut` | Same; per-tile transition, not orchestrated |
| End-state card appearance | `theme.motion.normal` (0.28s) | `.spring(response: 0.28, dampingFraction: 0.8)` | Subtle entrance ŌĆö calm, not celebratory; matches "calm-premium" brand |
| Difficulty switch (board reset) | `theme.motion.normal` (0.28s) | `.easeInOut` cross-fade | New board cross-fades old; no slide gimmick |
| Loss-state mine reveal | **Instant** in P3 (no animation) ŌĆö D-18 explicitly defers cascade to P5 | ŌĆö | P3 ships immediate flip on terminal-state transition; P5 layers stagger via `phase: MinesweeperPhase` |

**Reduce Motion (A11Y-03 ŌĆö Phase 5):** P3 ships `withAnimation(theme.motion.ease)` wrappers on transitions but does NOT implement the `accessibilityReduceMotion` dampening ŌĆö that is P5's surface. P3 verifies the wrappers exist so P5's switch is single-line.

## Gesture Composition (SC1 ŌĆö load-bearing)

```swift
// MinesweeperCellView body
Tile(cell: cell, theme: theme)
    .gesture(
        LongPressGesture(minimumDuration: 0.25)
            .exclusively(before: TapGesture())
            .onEnded { result in
                switch result {
                case .first:  vm.toggleFlag(at: index)   // long-press won
                case .second: vm.reveal(at: index)        // tap won
                }
            }
    )
```

**Why `.exclusively(before:)` and not `.simultaneously`:** prevents the "tap ŌåÆ reveal AND flag fired" misfire mode. The 0.25s threshold is the load-bearing constant ŌĆö too short causes tap-when-meant-to-flag, too long feels unresponsive. **Locked at 0.25s** per ROADMAP SC1.

**Validation gate:** 50 manual taps on iPhone SE-class hardware (320pt width) with zero misfires. Documented in `03-VERIFICATION.md` at phase end.
