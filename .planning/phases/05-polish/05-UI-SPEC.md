# Phase 05 ‚Äî UI Design Contract

> Visual and interaction contract for the Polish phase (animation + haptics + SFX + a11y + theme matrix + Settings spine + 3-step intro). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Stack note:** Native iOS 17+ / SwiftUI / Swift 6 project. **DesignKit** (local SPM at `../DesignKit`) fills the role shadcn fills on web; the shadcn / Tailwind preset format is structurally non-applicable. Phase 5 is the largest UI surface in the milestone ‚Äî net-new screens (`IntroFlowView`, `FullThemePickerView`), the rebuilt Settings spine (APPEARANCE / AUDIO / DATA / ABOUT), and the polished animation pass on the existing Mines surfaces. P5 ships **zero new DesignKit components** and **zero new tokens** ‚Äî the full surface is consumed read-only from the existing kit. Per CONTEXT 28 decisions are pre-locked; this spec writes prescriptions, not options.

## Spacing Scale

Declared values (DesignKit `SpacingTokens` ‚Äî already on disk at `../DesignKit/Sources/DesignKit/Layout/SpacingTokens.swift`):

| Token | Value | Usage in Phase 5 |
|-------|-------|------------------|
| `theme.spacing.xs` | 4pt | Section-header leading inset (matches P1+P4 `settingsSectionHeader` baseline preserved); intra-icon padding inside SF Symbol leading glyph for AUDIO toggle rows; horizontal hairline gap between TabView page indicator dots (system default, but defines our visual rhythm); `.symbolEffect(.bounce)` magnitude reference (system-handled) |
| `theme.spacing.s` | 8pt | Vertical rhythm between items inside a single Settings card group (between Haptics and Sound effects rows; between About rows); horizontal gap between SF Symbol leading glyph and label inside each AUDIO toggle row (matches P4 SettingsActionRow precedent); IntroFlowView Skip button trailing inset from safe area; sign-in card vertical stack rhythm between SIWA button and Skip button (D-21) |
| `theme.spacing.m` | 12pt | Vertical inner padding of each AUDIO toggle row (matches P4 tappable row baseline so total row height with `body` text + 44pt minHeight reads symmetric to P4 DATA section); IntroFlowView body copy paragraph rhythm; FullThemePickerView outer top inset between navigation title and the embedded `DKThemePicker(catalog: .all)` |
| `theme.spacing.l` | 16pt | `DKCard` outer padding (already baked into `DKCard` ‚Äî DKCard.swift line 14); IntroFlowView outer step-content horizontal screen-edge padding; vertical gap between IntroFlowView title block and the visual element below it (5-swatch row in Step 1, mock StatsCard in Step 2, sign-in card in Step 3); APPEARANCE section vertical gap between the 5-swatch DKThemePicker row and the "More themes & custom colors" NavigationLink row |
| `theme.spacing.xl` | 24pt | Vertical gap between **section** card groups in Settings (APPEARANCE ‚Üí AUDIO ‚Üí DATA ‚Üí ABOUT ‚Äî matches P1+P4 inter-section rhythm); IntroFlowView vertical gap between body copy block and the bottom-trailing Continue/Done button block (so the visual breathes on iPhone SE); FullThemePickerView gap between the embedded full-catalog grid and the bottom safe-area inset |
| `theme.spacing.xxl` | 32pt | IntroFlowView top-safe-area inset before the step title (so the title sits below the Dynamic Island / status bar without being cramped); IntroFlowView vertical gap between the visual element (swatches / mock card / sign-in card) and the bottom Continue/Done button on tall devices (auto-grows via `Spacer()`, but `xxl` is the minimum reserved breathing room) |

**Exceptions:**
- **Tappable Settings toggle row minimum height = 44pt** (Apple HIG, matches `DKButton` floor and P4 `SettingsActionRow` carve-out). Applied via `.frame(minHeight: 44)` on the row's outermost `HStack`. Per CLAUDE.md ¬ß1 / FOUND-07: bare `.frame(minHeight: 44)` is **not** flagged by the pre-commit hook (which scans `padding(<int>)` / `cornerRadius:` / `Color(`); minHeight is HIG-mandated and has no equivalent token).
- **IntroFlowView step-card maximum width = 480pt** for SIWA card and step-content blocks on iPad-class widths (`frame(maxWidth: 480, alignment: .center)`). Centered on iPad to avoid over-wide line lengths; passes through unchanged on iPhone widths (‚â§430pt).
- **Loss-shake horizontal magnitude = 8pt** (D-03). Applied via `.offset(x: shakeOffset)` where `shakeOffset` is keyframe-driven `0 ‚Üí +8 ‚Üí ‚àí8 ‚Üí +4 ‚Üí 0`. Magnitude is a behavior constant (animation amplitude), not a layout spacing value ‚Äî documented here so the executor uses the locked literal, not a token.
- **Win-wash overlay opacity peak = 0.25** (D-02). Behavior constant (alpha range), not a spacing value ‚Äî documented for the executor.
- **No other exceptions.** P5 introduces zero ad-hoc padding ‚Äî all paddings/insets read `theme.spacing.{token}`.

**Hard cap:** Zero raw `.padding(8)` / `.padding(.horizontal, 16)` / `.padding(.vertical, 12)` integer literals in `Screens/IntroFlowView.swift`, `Screens/FullThemePickerView.swift`, `Screens/SettingsView.swift`, or any P5-edited Mines view (`MinesweeperGameView`, `MinesweeperBoardView`, `MinesweeperCellView`, `MinesweeperEndStateCard`). Every padding call site reads `theme.spacing.{token}`. Verified by grep + the FOUND-07 hook.

```swift
// Flag spring

    .symbolEffect(.bounce, value: reduceMotion ? 0 : vm.flagToggleCount)
    // Tying value to constant 0 effectively disables the trigger

// Win sweep
.overlay(
    Rectangle()
        .fill(theme.colors.success.opacity(reduceMotion && vm.phase == .winSweep ? 0.25 : winWashAlpha))
        .allowsHitTesting(false)
        .ignoresSafeArea()
)
.phaseAnimator(reduceMotion ? [0.0] : [0.0, 0.25, 0.0], trigger: vm.phase == .winSweep) { content, phase in
    content.opacity(phase)
}

// Loss shake
.offset(x: reduceMotion ? 0 : shakeOffset)
.keyframeAnimator(initialValue: 0.0, trigger: reduceMotion ? nil : vm.phase) { ... }
```

The Reduce Motion contract is **tested via PreviewProvider environment override** (`.environment(\.accessibilityReduceMotion, true)`) for every animated surface, plus a manual SC5 audit ("Reduce Motion ON during a win replay produces a static end-state overlay with no shake or sweep").

   - User taps **Skip** (inside the sign-in card OR top-trailing ‚Äî both work) ‚Üí write `hasSeenIntro = true` ; dismiss.
   - User taps **Done** (bottom-trailing on Step 3) ‚Üí write `hasSeenIntro = true` ; dismiss.

### Settings APPEARANCE flow

1. User opens Settings tab ‚Üí sees APPEARANCE card with 5 inline Classic swatches + "More themes & custom colors" nav row.
2. User taps a swatch ‚Üí `themeManager.preset` updates instantly; the entire app re-renders with the new preset (DKThemePicker is wired internally ‚Äî view consumes `themeManager` directly).

4. In `FullThemePickerView`: full `DKThemePicker(catalog: .all)` segmented Presets / Custom; user picks a preset OR opens Custom tab and overrides anchors via 4 color wells; changes are live (commit-on-change via `themeManager.overrides` setter).

### Settings AUDIO flow

1. User taps the Haptics toggle ‚Üí `settingsStore.hapticsEnabled` flips; `didSet` writes to UserDefaults instantly (P4 pattern).
2. User taps the Sound effects toggle ‚Üí `settingsStore.sfxEnabled` flips; same.
3. Next gameplay tap/flag/win/loss event reads the live flag at the source (`Haptics.playAHAP` / `SFXPlayer.play` early-return if flag is false) ‚Äî no scene re-render needed, no view-layer wiring.

### Settings ABOUT flow

1. User sees Version row with `"1.0 (42)"` trailing ‚Äî read-only.
2. User taps Privacy row ‚Üí `withAnimation(.easeInOut(duration: theme.motion.fast)) { isPrivacyExpanded.toggle() }` ‚Üí inline body copy slides in / out below the row in the same DKCard. Chevron rotates to indicate state.
3. User taps Acknowledgments row ‚Üí NavigationLink push to `AcknowledgmentsView` (static text).

### Mines animation flow

1. **Pre-first-tap:** `MinesweeperPhase = .idle`. No animations active.
2. **First tap:** VM transitions `gameState = .playing` AND `phase = .revealing(cells: [firstTapIndex])` ; cascade animates the 1 revealed cell. Tap haptic fires via `.sensoryFeedback(.selection)`. Tap SFX fires via `SFXPlayer.play(.tap)` (gated). Phase resets to `.idle` after the cascade transition completes (no explicit timer needed ‚Äî SwiftUI's `.transition` handles cleanup automatically; VM may hold `.revealing` for the cascade duration to support tests but doesn't need to drive cleanup).
3. **Subsequent reveal (single cell or flood-fill):** Same ‚Äî VM sets `phase = .revealing(cells: revealedList)` ; cascade per D-01.
4. **Flag toggle:** VM increments `flagToggleCount` (Int counter). `.sensoryFeedback(.impact(.light))` fires on the cell. `.symbolEffect(.bounce, value: flagToggleCount)` triggers the flag spring. No phase transition ‚Äî flag is a per-cell event, not a board-wide one.
5. **Win:** VM transitions `gameState = .won` AND `phase = .winSweep`. Win haptic fires (`Haptics.playAHAP("win")`). Win SFX fires (gated). Win wash animates over `theme.motion.slow`. End-state DKCard fades in concurrently (P3 contract preserved).
6. **Loss:** VM transitions `gameState = .lost(mineIdx)` AND `phase = .lossShake(mineIdx)`. Loss haptic fires. Loss SFX fires (gated). Loss shake animates over 0.4s. Mine reveal cascade (per D-17 step 2 ‚Äî every other mine flips to revealed-as-mine glyph) ‚Äî preserved from P3 instant render in P3, **STILL instant in P5** (no per-mine cascade animation; the loss-shake IS the orchestrated motion). End-state DKCard fades in.
