<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Apple CloudKit Dashboard ↔ Production environment | The Deploy button writes record types into the Production environment that real users will sign in to. Reversible per record-type until first Production write; once a TestFlight user signs in and writes, additive-only. |
| Binary entitlement ↔ Dashboard container | `gamekit.entitlements` declares `iCloud.com.lauterstar.gamekit`. The Production schema must live under that container ID exactly — any drift = "Works in Xcode but not TestFlight" failure mode (Pitfall 11 warning). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-03-wrong-method | Integrity | Schema deploy method | mitigate | D-04 lock: USE the Dashboard button. Forbid `initializeCloudKitSchema()` against Production. Task 1 acceptance criterion checks the screenshot path (proves user used the Dashboard, not app code). |
| T-07-03-container-drift | Tampering | Container ID lock | mitigate | Task 3 grep-verifies `iCloud.com.lauterstar.gamekit` is unchanged in `gamekit/gamekit/gamekit.entitlements` and `gamekit.xcodeproj/project.pbxproj`. P1 pre-commit hook also enforces. |
| T-07-03-wrong-record-types | Loss of Availability | Production schema | mitigate | Task 2 [BLOCKING] CHECKPOINT requires the user to visually confirm `CD_GameRecord` AND `CD_BestTime` exist in Production with same indexes as Development (verification rung 1, D-06). |
| T-07-03-irreversible | Compliance / Reversibility | Production environment | accept | D-04 confirms reversibility per record-type until first Production write. SC3 sweep in 07-06 is the first Production write event; this plan is reversible until then. |
| T-07-03-pitfall-3 | Loss of Availability | Silent CloudKit sync failure | mitigate | Pitfall 3 + Pitfall 11: "CloudKit container not provisioned for Production — works in development sandbox, fails in TestFlight because Production environment is empty." This plan is the mitigation for that exact failure mode. |
</threat_model>
