## Decisions Made

1. **Inline-helper refactor over sibling-file split.** CONTEXT D-05's Claude's-Discretion block listed (a) extract into `NonogramGameView+SmallZone.swift`, (b) extract only if file > 500, (c) inline if delta ≤ 15 LOC. Initial delta crossed cap by 10; chose to factor the three repeated sub-views into local `private @ViewBuilder` helpers in the same extension rather than spawn a new file. Reasoning: helpers stay scoped to `smallZoneLayout` only (don't leak to `existingLayout` / `largeZoneLayout` byte-identity), file locality is preserved, and the pattern matches sibling helpers already living in the same extension (`compactRowComposed`, `restartWithOverflowMenu`).
2. **`anchors.picker` consumed implicitly via `headerBarAtBottom`.** D-09 + the 12.1-02/03 sibling-plan pattern: the 4 Small zones' (`headerBar`, `picker`) pairs collapse to exactly 2 VStack orderings. A separate `pickerAtBottom` boolean would have been dead-equivalent. Used a single boolean to match sibling-plan shape and minimize branching surface.
3. **Soft-reset over `git rm`/`git revert` for the cross-staged-files recovery.** Soft reset preserves worktree + index entries (non-destructive per the executor's `<destructive_git_prohibition>` boundary — that prohibition targets `git clean` / `git reset --hard` / blanket `git restore .` which DELETE files; `--soft` modifies only the branch ref). Cleanest recovery path that returns the sibling's files to their pre-my-add state.
