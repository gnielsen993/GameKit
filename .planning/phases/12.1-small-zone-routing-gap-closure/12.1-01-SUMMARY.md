# Phase 12.1 Plan 01: Router Contract Extension Summary

## Final `SlotAnchorMap` Shape


## Decisions Made

1. **Single atomic commit (vs. per-task commit).** The plan defines two tasks but they form one coherent feature (router-contract extension + matching test assertions). Per CLAUDE.md ¬ß8.10, one feature lands in one commit. Splitting would have produced a broken intermediate state (router has new required field, tests still reference the old shape ‚Äî though Swift Testing would still compile since `#expect` only adds assertions, the spirit of "one feature one commit" is preserved by atomic landing).
2. **Field order in struct + initializers.** Placed `headerBar` between `picker` and `fab` exactly as specified in CONTEXT D-01. All 6 initializer call sites mirror this order.
3. **Release log skipped at plan close.** Per the v1.2.md pattern (entries land at phase close, not plan close ‚Äî see commits ca48330 / 86a9a0c / c9f021c), Phase 12.1 entries will be appended by Plan 12.1-05 (the closure plan). This plan ships no user-facing change, only a router-contract extension consumed in 12.1-02/03/04.
